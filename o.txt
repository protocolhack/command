The current implementation uses SPKI-based pinning, meaning the pin is derived from the intermediate certificate’s public key, not the certificate itself. As a result, the app will continue to trust the connection even if the intermediate certificate is re-issued or its serial number or validity period changes, as long as the public key remains the same. This is secure but creates a broader trust scope than certificate-level pinning. With certificate pinning, any change to the intermediate certificate would break trust. We should either update the User Story to explicitly allow SPKI pinning or modify the implementation to pin the full intermediate certificate.
